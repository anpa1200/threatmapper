# AdversaryGraph v4.1.0 Release Notes

AdversaryGraph v4.1.0 is the Asset Attack Surface Mapping release.

## Added

- `POST /api/asset-surface/analyze` for asset inventory analysis.
- Asset inventory normalization for CSV, JSON, TXT, hostname/IP lists, CMDB
  exports, cloud inventory, scanner output, service lists, and mixed pasted
  text.
- Deterministic scoring for exposure, criticality, remote administration,
  web/API services, databases, identity services, cloud storage, containers,
  CI/CD, and legacy technology signals.
- ATT&CK candidate generation with technique IDs, names, rationales, confidence,
  priority actions, and validation steps.
- Optional AI enrichment that augments the deterministic baseline with executive
  summary, attack paths, control gaps, validation gaps, assumptions, and
  cross-asset findings.
- Asset Surface UI at `/asset-surface` with upload/paste controls, provider
  selection, AI toggle, risk/exposure filters, clickable IOC/TTP tags, JSON
  export, layer save, and Navigator handoff.
- Previous Analyses history in the browser for recent Asset Surface runs.
- White asset-surface comparison layer in Navigator for inventory-derived TTPs.
- Example inventory file: `asset-inventory-example.csv`.
- v4.1 screenshot set under `docs/assets/adversarygraph-v4.1-platform`.

## Changed

- Discover now exposes Asset Surface, Malware Analysis, String Analyzer,
  Decompilation and Debug, Malware Unpacker, and Dynamic Analysis launchers.
- Sidebar layout now supports long navigation lists without cutting off lower
  modules.
- Active version markers are updated to `4.1.0` in `VERSION`,
  `backend/app/core/version.py`, `frontend/package.json`,
  `frontend/package-lock.json`, and the UI footer.
- README and platform documentation now point to the v4.1 release summary,
  release notes, Asset Surface guide, and current screenshot manifest.

## Validation Limits

Asset Surface Mapping does not prove reachability or exploitability. Analysts
must validate exposure with scanner data, cloud firewall/security-group state,
WAF and VPN configuration, CMDB ownership, patch level, authentication controls,
and telemetry coverage before taking operational action.

## Tests

- Backend: `python -m pytest -o addopts='' tests/unit/test_asset_surface.py`
- Frontend: `npm run build`
