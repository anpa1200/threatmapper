# AdversaryGraph v3.0.0 Release Notes

Release date: 2026-06-20

AdversaryGraph v3.0.0 is the investigation workbench release. It moves the
platform beyond static ATT&CK mapping and IOC lookup into a practical workflow
for investigating suspicious artifacts, logs, PCAP-derived telemetry, and
source-backed relationship graphs.

## Major Changes

- Added Tier 3 IOC Investigation expansion for IPs, domains, URLs, hashes, and
  suspicious artifacts.
- Added saved IOC investigation sessions with history, reload, and delete
  actions.
- Added clickable relationship-graph node pages for every node type.
- Added graph focus mode: selecting a node can show only its direct connected
  neighborhood.
- Added observable reinvestigation from graph nodes, node lists, and node detail
  pages.
- Added an analyst-focused graph mode that hides noisy context labels by
  default while keeping raw context available through `All nodes`.
- Added evidence ranking based on source reliability, relation tier, directness,
  suspicious context, and corroborating relations.
- Added next-best pivot ranking to guide the analyst toward the most useful
  follow-up observables.
- Added timeline extraction from enrichment raw data.
- Added source-conflict summaries to make mixed benign/suspicious provider
  signals visible.
- Added urlscan activity analysis for suspicious page behavior patterns and
  ATT&CK technique leads.
- Expanded AI log and PCAP analysis workflows for extracting IOCs, suspicious
  activity, PowerShell/function/script artifacts, ATT&CK mappings, actor leads,
  and report-ready summaries.
- Improved AI report formatting for clearer analyst handoff.

## Investigation Workflow

The `/ioc-investigation` page now supports:

- source status and enrichment evidence
- Tier 1 / Tier 2 / Tier 3 pivots
- source-backed ATT&CK technique leads
- actor/APT leads with explicit caveats
- kill-chain/tactic coverage
- relationship graph focus mode
- clickable node details
- saved investigation history
- evidence ranking
- next-best pivots
- extracted timeline
- source-conflict review
- optional AI summary generation

## AI Log And PCAP Analysis

The AI Analysis workflow supports pasted telemetry or uploaded files such as
logs, investigation notes, IOC lists, and PCAP-derived text exports. The
analysis workflow can extract:

- IPs, domains, URLs, hashes, and other observables
- suspicious PowerShell, command-line, script, and function artifacts
- likely ATT&CK techniques
- actor-overlap hints
- suspicious/malicious activity explanations
- report-ready narrative output
- TTP layers that can be injected into Navigator for overlap review
