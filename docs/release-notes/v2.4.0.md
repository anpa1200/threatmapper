# AdversaryGraph v2.4.0 Release Notes

Release date: 2026-06-18

AdversaryGraph v2.4.0 improves self-hosted operations by making the reference
database dynamic and the deployment data layout explicit.

## Major Changes

- Added daily dynamic DB synchronization for:
  - MITRE ATT&CK Enterprise, Mobile, ICS, and ATLAS reference data
  - MISP Galaxy actor metadata
  - configured IOC intelligence sources
- Added an external persistent Postgres directory:

```env
ADVERSARYGRAPH_DB_DIR=./data/postgres
```

- Added sync scheduling controls:

```env
DYNAMIC_DB_SYNC_HOUR=3
DYNAMIC_DB_SYNC_MINUTE=30
DYNAMIC_DB_IOC_SYNC_DAYS=7
```

- Added manual dynamic DB refresh:

```text
POST /api/sync/dynamic-db
```

- Added a Reference Sync UI action for the same workflow.
- Added a migration helper:

```bash
./scripts/migrate-postgres-volume-to-external-dir.sh adversarygraph_pg_data
```

- Extended self-test details with database host/name and persistent data
  directory information.
- Fixed actor IOC count mismatch by aligning list and detail views on current
  active 180-day IOC counts.

## Data Model Intent

AdversaryGraph now treats public reference data and private analyst data as
separate operational concerns:

- Public reference data can be refreshed and rebuilt.
- Private reports, custom IOC feeds, custom IOCs, actor links, and analyst
  artifacts remain in the persistent external database directory.

The dynamic sync task is additive/update-oriented and does not intentionally
purge private analyst records.

## Upgrade Notes

For a fresh deployment:

```bash
git pull
cp .env.example .env
docker compose up -d --build
```

For an existing named-volume deployment, migrate the current Postgres volume
before relying on the external directory:

```bash
docker compose stop api worker beat frontend postgres
./scripts/migrate-postgres-volume-to-external-dir.sh adversarygraph_pg_data
docker compose up -d --build
```

Then verify:

```bash
curl http://localhost:8000/api/health
curl http://localhost:8000/api/system/selftest
```
