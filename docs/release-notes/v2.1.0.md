# AdversaryGraph v2.1.0 Release Notes

Release date: 2026-06-17

AdversaryGraph v2.1.0 adds sector relevance and IOC intelligence workflows on top
of the v2 self-hosted AI CTI platform. The release is focused on practical
client context: which actors matter for a sector, which TTPs are relevant to a
technology environment, and which current or historical IOCs can be linked to an
actor from public feeds or analyst-provided reports.

## Major Changes

- Sector Intelligence page for ranking actors by sector, region, technology or
  environment, activity window, TTP depth, and evidence quality.
- MISP Galaxy threat-actor sync for local sector, region, origin, motivation,
  alias, and evidence observations.
- IOC Intelligence data model for sources, indicators, and actor links.
- ThreatFox, AlienVault OTX, custom JSON/CSV/TXT feed sync, and manual IOC
  import support.
- Report-upload IOC extraction for private PDF/DOCX/TXT analyst inputs.
- Actor IOC tabs with freshness filtering, confidence/source evidence, counts,
  and CSV export.
- Centralized IOC sync controls in Reference Sync.
- Searchable A-Z multi-select filters for Sector Intelligence sectors, regions,
  and technologies/environments.
- Sector Intelligence shortcuts to actor profile, TTP profile, IOC view, and
  Navigator overlay.

## Analyst Impact

The release makes AdversaryGraph more useful for daily CTI prioritization:

- map client sectors and environments to relevant actors;
- move from relevant actors to relevant TTPs and Navigator coverage;
- enrich actor views with public or private IOCs;
- keep IOC source sync centralized and repeatable;
- preserve analyst control over uploaded private reports.

## Verification

- Frontend build: `npm run build` passed.
- Backend tests: IOC and sector integration tests, ATT&CK service unit tests,
  and full pytest suite were run during release preparation.

## Upgrade Notes

Use the normal Docker workflow:

```bash
git pull
cp .env.example .env
docker compose up -d --build
```

Optional IOC providers need local API keys or feed URLs in `.env`. Keep private
keys out of commits.
