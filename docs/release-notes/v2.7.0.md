# AdversaryGraph v2.7.0 Release Notes

Release date: 2026-06-20

AdversaryGraph v2.7.0 adds a dedicated IOC Investigation workflow. Analysts can
start from an IP, domain, URL, hash, or suspicious artifact, collect enrichment
from configured sources, expand Tier 1 and Tier 2 relationships, map ATT&CK
technique leads, identify actor/APT leads, and prepare an AI-ready investigation
summary without leaving the platform.

## Changes

- Added `POST /api/ioc/investigate` for IOC and artifact investigations.
- Added IOC type classification for IPs, domains, URLs, hashes, and fallback
  artifact searches.
- Added local IOC DB enrichment, including records imported from OpenCTI, MISP,
  STIX/TAXII, custom feeds, and reviewed reports.
- Added source pivots for VirusTotal, ThreatFox, MalwareBazaar, AlienVault OTX,
  urlscan.io, GreyNoise, AbuseIPDB, and Shodan.
- Added Tier 1 relationship expansion from enrichment source data and Tier 2
  local pivots for discovered IOC-like values.
- Added ATT&CK technique extraction from source metadata and mapped enrichment
  evidence.
- Added actor/APT lead extraction from local actor links and local actor alias
  matching across source tags, reports, malware names, and classification text.
- Added deterministic suspicion scoring and verdicts that discount benign
  reputation context and avoid treating generic text labels as malicious proof.
- Added optional AI summarization using the configured LLM provider.
- Added a new `/ioc-investigation` UI page with source status, relationship
  graph, TTP leads, actor leads, kill-chain/tactic context, and action buttons.
- Added Discover page shortcuts into IOC Investigation.
- Added environment and operator documentation for optional IOC Investigation
  provider keys.

## Upgrade

```bash
git pull
docker compose up -d --build
```

Optional provider keys can be added to `.env`:

```env
VIRUSTOTAL_API_KEY=
THREATFOX_AUTH_KEY=
OTX_API_KEY=
URLSCAN_API_KEY=
GREYNOISE_API_KEY=
SHODAN_API_KEY=
ABUSEIPDB_API_KEY=
CENSYS_API_KEY=
CENSYS_ORG_ID=
```

Leave optional keys empty when a provider is not required. The investigation
workflow reports missing keys per source instead of blocking the base platform.
