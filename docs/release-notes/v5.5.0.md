# AdversaryGraph v5.5.0 Release Notes

Release date: 2026-06-30

## Highlights

- Added enterprise access-control documentation and validation evidence for native authentication, RBAC, session administration, MFA workflow support, trusted proxy SSO metadata, and audit history.
- Updated the main product documentation, README, roadmap, security policy, Helm metadata, and version matrix to use v5.5.0 as the current release.
- Added screenshot-backed commercial trust material covering protected login, Admin Panel, observability, CVE Library, attack simulation, malware analysis, and IOC investigation workflows.
- Improved the platform sidebar, footer, feedback links, evidence graph entry points, and public documentation references.

## Validation

- Backend tests, backend lint, backend SAST, backend dependency audit, frontend build, frontend dependency audit, Docker build checks, container scanning, secret scanning, and version consistency are enforced by CI.
- The release includes reviewer-facing documentation for deployment, security posture, validation boundaries, demo data, malware-analysis boundaries, and SIEM forwarding safety.

## Upgrade Notes

- Native authentication is still configurable. Keep `AUTH_ENABLED=true` for exposed deployments.
- Existing local development deployments can continue to use `AUTH_ENABLED=false`, but they must not be exposed to untrusted networks.
- Review `.env.example`, `docker-compose.prod.yml`, and the authentication guide before a production deployment.
