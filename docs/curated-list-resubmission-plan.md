# Curated List Resubmission Plan

AdversaryGraph was previously declined from a curated awesome list because it did not yet show enough maturity, documentation, ownership transparency, quality evidence, age, and non-promotional signal.

This plan defines the evidence required before resubmission.

## Do Not Resubmit Until

- The project is at least 6-8 weeks older than the first submission.
- There are at least three tagged releases after the initial public release.
- CI is consistently green.
- Quickstart has been tested from a clean machine.
- Demo dataset and sample outputs are current.
- At least one external user has filed an issue, discussion, bug report, or installation feedback.
- Documentation includes limitations, security model, admin guide, and validation notes.

## Minimum Evidence Package

| Requirement | Evidence |
|---|---|
| Maturity | Release history, changelog, roadmap |
| Documentation | README, quickstart, user guide, admin guide, limitations |
| Maintenance | Maintainers file, issue templates, security policy |
| Quality | CI, tests, sample outputs, validation plan |
| Adoption | External issues, forks, stars, mentions, or users |
| Fit | Short neutral description aligned with CTI tools/resources |

## Resubmission Wording

Use neutral wording:

```text
Adds AdversaryGraph, a self-hosted CTI-to-detection workbench for ATT&CK mapping,
TTP-overlap comparison, detection-gap review, and analyst-reviewed exports.

Maturity evidence:
- Tagged releases and changelog
- Docker quickstart and self-hosted deployment guide
- CI-tested backend and frontend
- Demo dataset and sample outputs
- Security model and limitations documented
- Validation plan and mapping-review rubric
```

Avoid:

- Marketing language.
- Multiple links to the same personal ecosystem.
- Claims of adoption without evidence.
- Reopening immediately after rejection.

## Suggested Timing

Target resubmission at v5.0.0 or later. The project is now stable and past the initial public release maturity bar.
